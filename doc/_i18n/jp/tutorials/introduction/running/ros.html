前提条件

mc_rtc_tickerを使用してROS内でコントローラーを実行するには、ROSのサポート付きでmc_rtcをビルドする必要があります。また、{% link mc_rtc_ros %}も必要です。

ここでは、roscoreが既に実行されているか、ROS_MASTER_URIがアクティブなマスターを指していることを前提に説明します。

表示インターフェイスを実行する

以下のコマンドを実行します。すると、mc_rtcの専用パネルと使用するロボットのモデルが読み込まれた状態でRVizが起動します。

{% highlight bash %} $ roslaunch mc_rtc_ticker display.launch {% endhighlight %}

2台以上のロボットをシミュレートする場合は、新しいロボットモデルをシーンに追加する必要があります。ロボットに関する記述は、パラメーター/control/env_Nにパブリッシュされます。ここで、Nはロボットのインデックスです。

コントローラーを実行する

コントローラーを実行するには、以下のコマンドを実行します。

{% highlight bash %} $ rosrun mc_rtc_ticker mc_rtc_ticker {% endhighlight %}

なお、表示インターフェイスを再起動せずにコントローラーを再起動することができます。
